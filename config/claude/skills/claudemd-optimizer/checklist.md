# CLAUDE.md Optimization Checklist

CLAUDE.mdファイルを最適化する際のチェックリスト。Claude 4ベストプラクティスに基づく。

## 必須項目

### 明示性 (Explicitness)
- [ ] 指示が具体的で曖昧さがない
- [ ] 修飾語を使って期待レベルを明示（"as many as possible", "comprehensive", "fully-featured"）
- [ ] 成功基準が定義されている

### コンテキスト (Context)
- [ ] 各ルールに「なぜ」の説明がある
- [ ] プロジェクトの目的・背景が明記されている
- [ ] 技術スタック・アーキテクチャの概要がある

### 過剰エンジニアリング防止
- [ ] "Avoid over-engineering"セクションがある
- [ ] 不要な機能追加を抑制する指示がある
- [ ] 「必要最小限」の原則を明示

## 推奨項目

### ツール使用パターン
- [ ] デフォルトの行動モード（proactive/conservative）を指定
- [ ] コード探索の徹底を指示（読む前に提案しない）
- [ ] 並列ツール実行の推奨

### 出力形式制御
- [ ] 「何をしないか」より「何をするか」で指示
- [ ] 必要に応じてXMLタグで形式を指定
- [ ] Markdown使用の方針を明示

### コード品質
- [ ] テスト駆動の方針（ある場合）
- [ ] ハードコード禁止の明示
- [ ] 一般的・再利用可能な解決策を推奨

### ハルシネーション防止
- [ ] コードを読む前に回答しない指示
- [ ] 推測を避ける指示
- [ ] 不明点は質問するよう指示

## プロジェクト固有

### 構造
- [ ] セクションが論理的に整理されている
- [ ] 重要な情報が先頭に配置されている
- [ ] 長すぎない（必要な情報のみ）

### 開発コマンド
- [ ] よく使うコマンドがリストされている
- [ ] テスト実行方法が明記
- [ ] ビルド・デプロイ手順がある

### コーディング規約
- [ ] Lint/Format設定への言及
- [ ] ファイル配置規約
- [ ] 命名規則

## アンチパターン（避けるべきこと）

| アンチパターン | 改善方法 |
|--------------|---------|
| 曖昧な指示（「良いコードを書く」） | 具体的な基準を明示 |
| 理由なきルール（「XXXを使わない」） | なぜかを説明 |
| 過度な冗長性 | 必要十分な長さに |
| ネガティブ指示のみ | ポジティブ指示に変換 |
| 汎用的すぎる | プロジェクト固有の情報を追加 |
| スキル/コマンドへの言及なし | 利用可能なスキルを紹介 |

## 最適化の優先順位

1. **Critical**: 過剰エンジニアリング防止、コード探索の徹底
2. **High**: 明示的な指示、コンテキスト追加
3. **Medium**: 出力形式制御、ツール使用パターン
4. **Low**: 細かいフォーマット調整

## 評価基準

最適化後のCLAUDE.mdは以下を満たすべき:

- Claudeが自律的に作業可能
- 追加質問なしで意図が伝わる
- プロジェクトの文脈を理解できる
- 適切なツールを選択できる
- 過剰な実装を避けられる
